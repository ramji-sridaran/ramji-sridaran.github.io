│                        🎓 INTERVIEW TALKING POINTS                             │
├────────────────────────────────────────────────────────────────────────────────┤
│                                                                                │
│  1. Why Serverless over Traditional Servers?                                  │
│     ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                                │
│     • No server maintenance or patching required                              │
│     • Automatic scaling from 0 to millions of requests                        │
│     • Pay only for actual usage (not idle capacity)                           │
│     • Built-in high availability and fault tolerance                          │
│     • Perfect for portfolio site with variable traffic                        │
│                                                                                │
│  2. Cost Optimization Strategy                                                │
│     ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                                            │
│     • Maximize free tier usage (Groq: 14.4K requests/day)                     │
│     • Use paid backup only when free tier exhausted                           │
│     • Static fallback for complete provider failures                          │
│     • Result: $0-5/month even with significant traffic                        │
│                                                                                │
│  3. Reliability & Fault Tolerance                                             │
│     ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                                         │
│     • Three-tier fallback ensures zero downtime                               │
│     • Primary (99.5%) → Backup (99.9%) → Fallback (100%)                     │
│     • Combined availability: 99.95%+                                          │
│     • Graceful degradation - never show error to users                        │
│                                                                                │
│  4. Security Best Practices                                                   │
│     ━━━━━━━━━━━━━━━━━━━━━━━━━                                                │
│     • API keys stored in encrypted environment variables                      │
│     • Never exposed to frontend/client                                        │
│     • CORS protection prevents unauthorized access                            │
│     • Error sanitization - no internal errors leaked to users                 │
│                                                                                │
│  5. Real-World Trade-offs                                                     │
│     ━━━━━━━━━━━━━━━━━━━━━━━━━━                                               │
│     • Cold start delay (~200ms) vs no server maintenance                      │
│     • Free tier limits vs paid reliability                                    │
│     • Response time (~500ms) vs quality of AI responses                       │
│     • Complexity of fallback logic vs guaranteed availability                 │
│                                                                                │
